Funcionamiento básico de Angular

¿Sobre qué actúa Angular?

Angular actúa principalmente sobre componentes y plantillas HTML. Cada componente representa una parte de la interfaz de usuario, y la aplicación se construye combinando muchos componentes pequeños.

En aplicaciones de una sola página (SPA), como la de este tutorial, Angular no recarga la página completa, sino que va actualizando las vistas dentro de la misma página según la ruta seleccionada.

Componentes, plantillas y rutas

En este tutorial se ha usado la siguiente estructura:

  • Un componente raíz AppComponent, definido en app.ts, cuya plantilla principal es app.html. En esta plantilla se define la cabecera, el menú de navegación y un <router-outlet> donde se carga el contenido.
  • Varios componentes independientes (Inicio, Justificación, Instalación, etc.), cada uno con su propio archivo TypeScript y su propia plantilla HTML.
  • Un archivo de rutas app.routes.ts que indica qué componente se debe mostrar para cada ruta, por ejemplo /justificacion o /ejemplo.

Cuando el usuario hace clic en un enlace del menú con routerLink, Angular actualiza la ruta y carga el componente correspondiente dentro del router-outlet.

¿Cómo se editan y modifican los ficheros?

Los ficheros más importantes que se modifican al trabajar con Angular son:

  • Los archivos .ts de cada componente, donde se define la lógica y los datos que se quieren mostrar.
  • Los archivos .html de las plantillas, donde se coloca la estructura visual y se usan las expresiones de Angular para mostrar datos (por ejemplo, usando interpolación con dobles llaves), directivas, etc.
  • Opcionalmente, los archivos .css de cada componente o el styles.css global para definir la apariencia.

Cualquier cambio en estos archivos se refleja automáticamente en el navegador mientras el servidor de desarrollo está ejecutándose con ng serve.

¿Cómo se integran con otras aplicaciones?

En este tutorial no se ha utilizado ninguna API externa ni base de datos, pero Angular permite comunicarse con otras aplicaciones a través de peticiones HTTP (por ejemplo, a una API REST). Para ello se suelen crear servicios en TypeScript que se encargan de llamar a la API y proporcionar los datos a los componentes.

Aunque este proyecto es sencillo, la estructura básica que se ha utilizado (componentes, rutas, plantillas y estilos) es la misma que se utiliza en aplicaciones de Angular más complejas que sí consumen datos externos.

¿Cómo se referencia Angular desde otras aplicaciones?

Aunque este proyecto no utiliza servicios externos, Angular permite integrarse con otras aplicaciones mediante peticiones HTTP. Esto suele hacerse creando servicios que actúan como punto intermedio entre la aplicación y la API.

Un servicio puede realizar peticiones a una API REST (por ejemplo, obtener datos de usuarios), procesar la respuesta y entregarla a los componentes para su visualización. De esta forma, Angular puede formar parte de una arquitectura mayor compuesta de múltiples aplicaciones conectadas entre sí.
